So, when the variable is not yet assigned in bash, you could assign it when you call the bash script

example:
[root@scriptbox scripts]# ./5_args_websiteSetup.sh https://www.tooplate.com/zip-templates/2131_wedding_lite.zip 2131_wedding_lite

up there, the first variable is the link, and the second variable is the the zip name

$0 is the name of the script.
$1-$9 are the command line arguments.

echo $?					- print exit status of last command


[root@scriptbox scripts]# SEASON="Monsoon"
[root@scriptbox scripts]# export SEASON		- This will store the variable to only root user (cause we're in root)
Note: it'll be the same if we add the variable to /.bashrc or /.profile (means it'll only stored to one user not globally)

vi /etc/profile -> export "SEASON"			- This will store the variable to all users (global)

READ THE SUMMARY BLUD!!!!
To export variable for user we put export variable command in ~/.bashrc or ~/.profile file
To export variable for all users we put export variable command in /etc/profile file

read -p			- for the prompt [just like prompt() in javascript]
read -sp		- -s to hide the character when fill the prompt
----

echo "Your $SKILL is in high demand in the IT Industry."

echo

echo "Login successfull: Welcome USER $USR."
